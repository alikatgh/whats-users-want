Counting method

How the numbers are counted

Every number starts from the same support export. The dashboard counts support records, then groups them by the user goal they describe.

Analysis-ready records

Rows left after removing summary rows and helper columns. This is the base set used for the dashboard.

Tickets read by AI

High-signal records read by the local Mistral model to extract what the user wanted.

Discovered wants

Repeated goals found inside the AI-read tickets, such as access recovery or diamond transaction issues.

Mapped records

All analysis-ready records assigned to the closest discovered want using multilingual text embeddings.

Timeline numbers

For each month, count records mapped to that want. Partial final months are hidden by default.

Keyword numbers

Literal text matches only. For example, “restore” counts rows where that exact word appears.

No-ban records

Rows where the support text contains no-ban, no ban, or no_ban.

Repeat users

UIDs that appear more than once. Their rows are ordered by date to show the user journey.

Keyword vs want count

A keyword count asks: “does this exact word appear?” A want count asks: “is the user trying to do this thing?” That is why access recovery can be larger than the literal word restore.
